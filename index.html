ESP Web Flasher

Flashe dein ESP-Board direkt aus dem Browser – ohne esptool-Installation, ohne Kommandozeile. Board anstecken, auswählen, fertig.

Dieser Browser unterstützt kein Web Serial. Bitte öffne diese Seite in Google Chrome, Microsoft Edge, Opera oder Brave (Desktop). Firefox geht ebenfalls – dafür einmalig die WebSerial-Erweiterung firefox-webserial installieren (oder Firefox Nightly 151+ mit aktiviertem Web-Serial-Flag). Safari funktioniert nicht.

Voraussetzungen

  • Browser: Chrome, Edge, Opera oder Brave (Desktop) – Web Serial nötig. Firefox mit der Erweiterung firefox-webserial (oder Nightly 151+ mit Flag).
  • Kabel: ein echtes USB-Datenkabel (nicht nur Ladekabel).
  • Treiber / serielle Schnittstelle:
    • Heltec Wireless Paper: CP2102 USB-UART – unter macOS & Windows i. d. R. bereits vorinstalliert (Port z. B. /dev/cu.usbserial-… bzw. COMx).
    • Heltec Vision Master E213: kein USB-UART-Chip, sondern der native USB-Serial/JTAG des ESP32-S3 – kein Treiber nötig (Port z. B. /dev/cu.usbmodem… bzw. COMx). Dieser Port re-enumeriert auf macOS im Betrieb gelegentlich kurz, wodurch ein normaler screen-Monitor abbricht – dafür gibt es das Tool e213_monitor.py mit Auto-Reconnect (unten zum Download).
  • Verbindung: diese Seite muss über HTTPS geladen sein (auf craith.cloud gegeben).

🖥️ Serielle Konsole – Vision Master E213

Der native USB-Serial/JTAG-Port des E213 re-enumeriert auf macOS im Betrieb gelegentlich kurz – ein normaler screen-Monitor bricht dann ab. e213_monitor.py verbindet automatisch neu und überbrückt den Aussetzer (Befehle senden: Zeile tippen + ENTER; beenden mit Ctrl-C). Benötigt Python 3 + pyserial.

Experten-Modus · eigene Firmware & Konsole
Dieser Browser unterstützt kein Web Serial – der Experten-Modus ist deaktiviert. Siehe Browser-Hinweis oben.

Power-User-Werkzeug auf Basis von esptool-js: Chip-Infos auslesen, eine beliebige .bin an einem gewählten Offset flashen und die serielle Ausgabe mitlesen.

1 · Verbinden & Hardware erkennen

Beim Verbinden werden die sicheren Chip-Daten (Chip-Familie, Flash, PSRAM) ausgelesen und daraus die möglichen Module eingegrenzt. Teilen sich mehrere Boards denselben Chip, hilft die serielle Konsole (Schritt 3) bei der eindeutigen Erkennung.

Hinweis nRF52-Boards (RAK4631, Lilygo T-Echo, Heltec T114): Diese sprechen kein esptool-Protokoll und können hier nicht erkannt oder geflasht werden – sie werden per UF2-Drag&Drop bzw. ZIP-Bootloader programmiert.

2 · Eigene Firmware flashen

Offset beachten: Falscher Offset führt zu einem nicht bootenden Board. Im Zweifel ein Komplett-Image (0x0) verwenden. Der Chip wird automatisch erkannt. 



        
        
3 · Serielle Konsole & eindeutige Board-Erkennung

        

          Liest die Ausgabe der laufenden Firmware. Läuft bereits MeshCom,
          fragt „ℹ Board abfragen“ per --info die
          HWID ab – damit ist das Modul eindeutig bestimmt
          (Foto oben wird bestätigt). Bitte vorher oben „Stoppen“,
          falls noch verbunden – der Port kann nur einmal gleichzeitig
          genutzt werden.
